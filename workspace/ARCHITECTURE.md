# OpenClaw 多 Agent 架构说明

## 系统概览

OpenClaw 采用**主 Agent 编排 + 子 Agent 执行 + 独立评估器**的三层架构。主 Agent 不执行任何实际任务，只负责规划、路由和质量控制；子 Agent 各司其职，专注于特定领域；评估器（Vera）作为独立裁判，评审复杂任务的质量。

---

## 核心角色

### 主 Agent（Orchestrator）
- **职责：** 规划、路由、协调、呈现
- **输入：** 用户需求
- **输出：** 最终结果（或重做指令）
- **特点：**
  - 只给方向（What / Boundaries / Done Criteria），不给路径（How）
  - 通过任务路由表匹配最合适的子 Agent
  - 发起复杂任务评估，根据评分决定采用/改进/重做
  - 保留直连管道的编排权（什么时候可以跳过中转）

### 子 Agent（Executors）
目前已配置的 agent：
- **analyst（Mira 🔍）** — 市场情报 + 通用调研 + 数据分析，快速、精准、有来源
- **advisor（Quants 📊）** — 投资分析，量化评分
- **reporter（Scribe）** — 报告生成，格式化输出
- **content（Draft ✍️）** — 内容创作 + 翻译 + 视频制作
- **community（🌏）** — 社区运营 + 售后客服（印尼/东南亚市场）
- **evaluator（Vera ⚖️）** — 质量评估，独立打分
- **converter（🔄）** — 文件转换与内容提取
- 其他：seo, survey, legal, logistics, image, video, email, campaign, brand, crm, paid 等

**子 Agent 特点：**
- 高度专业化，边界清晰
- 自主决策执行路径
- 可以直接 handoff 给预授权的下游 Agent
- 定期整理 MEMORY.md，积累工作经验

### 评估器（Vera ⚖️）
- **职责：** 复杂任务的质量评估和打分
- **独立性：** 平行于主 Agent，不受其影响
- **打分机制：**
  - 7个任务类型模板（research / analysis / creative / investment / compliance / report / general）
  - 每个类型的权重不同（如合规类准确性50% vs 创意类创新性30%）
  - 根据维度分项打分，必须引用具体输出内容作为依据
- **记忆系统：** MEMORY.md 中保留各 Agent 的失误模式，后续评估时主动对照识别重复问题
- **日志：** 每次评估结果自动记录到 `~/.openclaw/workspace/memory/evaluation-logs.md`

## 工作流程

### 简单任务（主 Agent 直接处理）
```
用户请求 → 主 Agent 判断（不需要子 Agent）→ 直接回答 → 完成
```

### 单 Agent 任务
```
用户请求 → 主 Agent 判断 + 路由 → 子 Agent 执行 → 结果直接呈现 → 完成
```

### 复杂任务（多 Agent 协作）
```
用户请求
  ↓
主 Agent 判断（是否复杂）→ YES
  ↓
规划（三要素：目标、约束、交付标准）
  ↓
分配 → 子 Agent 1, 2, ... 执行
  ↓
汇总 → 打包评估包
  ↓
发送给 Vera（evaluator agent）
  ↓
Vera 根据任务类型选权重模板，分维度打分
  ↓
评分 ≥ 75 ？
  ├─ YES → 直接采用（或标注改进点）
  ├─ NO  → 返回子 Agent 重做（最多2次）
  └─ <60 → 向用户说明，呈现最佳结果
  ↓
完成
```

### 预授权直连管道（跳过主 Agent 中转）
上游 Agent 可直接 handoff 给下游：
- analyst → reporter（报告生成）
- analyst → advisor（投资分析）
- analyst → content（营销创作）
- advisor → reporter（投资报告输出）
- content → email（内容到邮件序列）
- content → video（内容到视频制作）
- content → image（内容到配图生成）
- data → reporter（数据到报告）
- data → analyst（数据支持分析）
- researcher → reporter（研究到报告）
- community → analyst（用户反馈到市场分析）
- seo → content（关键词到内容）
- brand → content（品牌到内容落地）

**规则：** 必须标注 `[直连移交 → agentId]`、打包完整信息、说明原始需求。

## 权重模板（Vera 评估时使用）

### research（情报收集）
- 准确性 40% — 来源可信？矛盾披露？
- 完整性 25% — 关键点全覆盖？
- 可操作性 20% — 下游 agent 好用？
- 创新性 5% — 非显而易见的数据？
- 表达质量 10% — 来源标注清晰？

### analysis（深度分析）
- 准确性 35% — 数据支撑充分？逻辑严谨？
- 完整性 25% — 各维度（SWOT/PEST）完整？
- 可操作性 25% — 结论落地为具体建议？
- 创新性 10% — 超出常规框架的洞察？
- 表达质量 5% — 逻辑结构清晰？

### creative（创意内容）
- 准确性 15% — 品牌信息/事实准确？
- 完整性 10% — 满足平台要求？
- 可操作性 25% — 可直接发布或只需微调？
- 创新性 30% — 吸引力强？突破同类？
- 表达质量 20% — 语言流畅自然？

### investment（投资分析）
- 准确性 45% — 数据权威？计算正确？风险披露？
- 完整性 25% — 基本面/技术面/宏观面全覆盖？
- 可操作性 20% — 有明确买入/卖出/观望建议？
- 创新性 5% — 独特量化视角？
- 表达质量 5% — 结论清晰呈现？

### compliance（法律合规）
- 准确性 50% — 法规引用正确？结论严谨？
- 完整性 30% — 覆盖所有风险点？
- 可操作性 15% — 建议具体到可执行步骤？
- 创新性 0% — 不适用
- 表达质量 5% — 无歧义、表述严谨？

### report（正式报告）
- 准确性 25% — 数据准确？来源标注？
- 完整性 30% — 目录结构完整？核心章节到位？
- 可操作性 15% — 结论清晰，读者可直接行动？
- 创新性 5% — 超预期的洞察/呈现？
- 表达质量 25% — 格式专业？语言流畅？

### general（通用）
- 准确性 30%
- 完整性 25%
- 可操作性 20%
- 创新性 15%
- 表达质量 10%

## 记忆系统

### 主 Agent MEMORY.md
- 空白模板，运行时逐步填充
- 记录用户偏好、技能安装、经验教训、待处理事项

### 子 Agent MEMORY.md
- **三部分：** 身份快照 / 工具箱 / 工作经验
- **只写这些：** 工具踩坑、搜索技巧、行业基准、流程改进
- **不写这些：** 每次任务结果、常规结论
- **内务整理（heartbeat 触发）：** MEMORY.md 超过 ~200 行时执行合并重复、清理过时、抽象提炼

### 评估器 MEMORY.md
- **权重模板表：** 7个任务类型模板，可根据使用反馈调整权重
- **失误模式记录：** 各 Agent 的重复性问题（同类问题 2 次以上才记录）
- **模板校正记录：** 权重调整历史
- **校正笔记：** 评分标准的系统性偏差

## 复杂任务规划纪律

主 Agent 规划时**必须且仅包含**：

| 要素 | 说明 | 示例 |
|------|------|------|
| **目标** | 这个任务要达成什么结果 | "输出一份市场竞争格局分析" |
| **约束** | 不能做什么/截止时间/质量底线 | "只看近6个月；不做财务预测" |
| **交付标准** | 什么样的输出算完成 | "包含定价、渠道、用户口碑三维度" |

**严禁出现：**
- ❌ 具体搜索关键词
- ❌ 工具使用指令
- ❌ 执行步骤顺序
- ❌ 方法论选择

路径由子 Agent 自主决定。

## 架构的核心设计原则

1. **编排 ≠ 执行** — 主 Agent 不做实际工作，只做规划和编排
2. **专业化边界** — 每个子 Agent 知道自己的能力和边界，超出的明确移交
3. **评估独立性** — 评估器（Vera）平行于主 Agent，防止自我确认偏差
4. **记忆积累** — MEMORY.md 存储工作经验，帮助 Agent 逐次改进
5. **预授权直连** — 高频协作的 Agent 对可以跳过主 Agent，提高效率
6. **质量把控** — 复杂任务必须评估，不达标返回重做（最多2次）
7. **明确路由** — 任务路由表消除主 Agent 的选择歧义
8. **客户隔离** — 客户会话（customer_*）的数据独立，不污染主 agent 记忆

## 扩展建议

1. **新增子 Agent：** 按现有 workspace-analyst 的结构（SOUL/AGENTS/MEMORY/TOOLS）创建，定义清晰的边界和移交规则

2. **权重调整：** 通过 Vera 的 evaluation-logs.md 定期回顾，找到系统性的评分偏差，在 Vera 的 MEMORY.md 中调整权重模板

3. **直连管道扩展：** 发现新的高频协作 Agent 对时，在主 Agent AGENTS.md 中新增到预授权清单

4. **Memory 管理：** 定期审视各 Agent 的 MEMORY.md 大小和质量，确保记录的是经验而非原始日志

---

_这个架构设计的核心是**分工明确、权力制衡、记忆积累**。通过让每个 Agent 做好自己擅长的事，同时通过独立评估器防止质量下滑，让 OpenClaw 逐次改进。_
